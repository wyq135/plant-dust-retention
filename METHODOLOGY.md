## 一、文献搜索

### 1.1 四个已纠正的系统性错误

| # | 错误 | 根因 | 纠正 |
|:--:|------|------|------|
| 1 | **系统性忽略硕博论文** | 搜索式未区分文献类型，默认结果排序偏期刊 | 学位论文专项搜索：`site:cnki.net 硕士论文`，万方学位论文库 |
| 2 | **把知网付费墙固化为硬约束** | 混淆了 Claude WebFetch 的 .cn 限制和用户实际网络环境 | 确认用户电脑接入大学内网，有完整下载权限 |
| 3 | **搜索维度单薄** | 仅按"物种名 + 滞尘"搜索 | 扩展为五维度矩阵（见下） |
| 4 | **数据库覆盖不完整** | 仅 WoS + 知网，遗漏万方 | 增加万方、维普、百度学术 |

### 1.2 五维度搜索矩阵

对每个目标物种必须执行：

| 维度 | 搜索式模板 | 示例 |
|------|------|------|
| 基础 | `{物种名} + 滞尘/颗粒物/降尘` | `香樟 滞尘` |
| 机构 | `{机构名} + 滞尘/颗粒物` | `华中农业大学 滞尘` |
| 导师/作者 | `{已知核心作者} + 滞尘` | `朱春阳 颗粒物` |
| 学位论文 | `{城市} + 植物/绿化 + 滞尘 + 硕士/博士` | `武汉 绿化 滞尘 硕士` |
| 高频期刊 | `{期刊名} + 滞尘 + {城市}` | `生态学报 滞尘 武汉` |

### 1.3 数据库优先级

```
P0: 知网 CNKI（含硕博论文库 CDMD）+ 万方数据
P1: Web of Science（国际期刊）
P2: 维普 CQVIP + 百度学术 xueshu.baidu.com
P3: Google Scholar
```

### 1.4 引用网络追踪

每篇纳入论文必做三项检查：
- **参考文献列表** → 发现前置研究
- **施引文献**（谁引用了它）→ 发现后续研究
- **同导师/同机构论文** → 发现团队系列研究

### 1.5 文献类型权重（数据密度排序）

```
硕士论文 > 博士论文 > 中文核心期刊 > 普通期刊 > 会议论文
```

单篇硕士论文可贡献 5-20 条记录（有完整实验设计章节和原始数据表格），而期刊论文常因篇幅限制压缩数据。

### 1.6 功能区位词扩展

论文标题和关键词中可能不写"功能区"，搜索时需涵盖所有具体场所词：

```
道路 | 交通 | 街道 | 工业区 | 工厂 | 公园 | 校园 | 绿地 | 居住区 | 小区 | 清洁区 | 郊区 | 立交桥 | 高架
```

## 二、PDF 文本提取与数据恢复

### 2.1 PM 标签丢失与恢复

**问题**：pdfplumber 提取中文 PDF 时，上下标（PM>10、PM10、PM2.5）的标签数字会丢失或跑到单独一行，原文中的 "PM10" 变成只剩 "PM"。

**恢复方法**（以 Chen Yazhen 2023 为例）：
1. 识别正文行中的孤立 "PM" 提及（如 `tion of PM (0.1426 g/m2) and PM (3.5865 g/m2)`）
2. 查看紧邻的下一行是否有关键数字（如 `10 >10 2.5`）
3. 将数字按顺序对齐，恢复为 `PM>10 (0.1426 g/m²)`, `PM10 (3.5865 g/m²)`
4. **用数学一致性验证**：恢复后的各级 PM 之和应等于 TSP 总量

### 2.2 中文 PDF 文本的常见失真

| 失真类型 | 示例 | 应对 |
|------|------|------|
| 词语粘连 | "单位叶面积滞尘量" → "单位叶面积滞尘量" | 人工断句 |
| 上下标丢失 | PM₁₀ → PM, PM₂.₅ → PM | 对齐行号恢复（见 2.1） |
| 表格行列错位 | 表头和数据对不齐 | 按数值大小逻辑匹配 |
| 物种名缩写 | 二球悬铃木 → PA, 香樟 → CC | 查找论文中的缩写定义表（通常在方法部分） |
| 中英文混排错乱 | 英文摘要插入中文段落 | 跳过，英文数据从英文部分重新提取 |

### 2.3 编码问题

- pdfplumber 输出使用 **UTF-8** 保存（不要用 GBK）
- CSV 文件统一使用 **UTF-8-sig**（BOM 头），确保 Excel 直接打开不乱码
- Windows 终端打印中文时，Python 脚本开头加：
  ```python
  import sys, io
  sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')
  ```

## 三、柱状图数据的现实局限

### 3.1 核心结论：柱状图 ≠ 可用数据

**柱状图如果没有精确的数字标注在柱顶或柱内，肉眼无法可靠读数。** 这不是阅读者的问题——学术论文发表时就没打算让读者从柱状图反推原始数据。

### 3.2 柱状图可以给的信息（勉强可用）

- **排序**：A > B > C > D（可验证数值合理性）
- **数量级**：都在 0.1-10 还是 10-100 范围
- **相对倍数**：A 约是 B 的 2 倍（不可靠，仅参考）

### 3.3 正文零星数据：意外的救命稻草

作者在讨论某个观点时，往往会引用具体数值。这些反而是**最可靠的数据**——经过作者自己确认。搜索方法：

```bash
# 在文本文件中搜索物种名附近的数值
grep -n "(香樟|桂花|海桐|石楠|广玉兰).*?\d+\.?\d*" paper.txt

# 搜索单位模式
grep -n "\d+\.?\d*\s*(g/m²|g/m2|g·m|μg/cm)" paper.txt

# 搜索英文物种名
grep -n "(camphora|fragrans|tobira|chinense|fraseri).*?\d+\.?\d*" paper.txt
```

**实际案例**：
- Changsha overpasses：正文精确提到麦冬 = 1.59 g/m²，但其他 8 个目标物种只在柱状图中 → 仅收获 1 条
- Sheng 2024：正文给二球悬铃木 = 0.951 g/m²，但中间排序的 6 个物种无精确值 → 仅收获 1 条

### 3.4 优先级判断

拿到一篇新论文，按以下顺序找数据：

1. **正文中的数值**（"X 物种的滞尘量为 Y g/m²"）→ 最可靠
2. **数据表格**（Table 1/2/3...）→ 最完整
3. **带数字标注的柱状图**（柱顶有数值）→ 可靠，但罕见
4. **无标注柱状图** → 不可用
5. **仅摘要/定性描述** → 不可用

## 四、测量方法与数据兼容性

### 4.1 方法学差异识别

不同论文使用不同的颗粒物洗脱方法，数值差异可达 **5-10 倍**：

| 方法 | 原理 | 数值特征 | 代表论文 |
|------|------|------|------|
| 超声清洗 + 滤膜称重 | 物理震荡剥离全部颗粒 | 标准值，可比性强 | 王琴 2020, 李海梅 2021 |
| 水溶消解 + 滤膜分级 | 化学消解 + 分级过滤 | **偏低 5-10 倍**，PM2.5 占比异常高 | Chen Yazhen 2023 |
| 刷洗法 | 物理刷洗 | 偏低，细微颗粒洗脱不足 | 少数早期论文 |

**强制规则**：不同方法的数据在统计分析时必须分层标注，不能混入同一模型。

### 4.2 方法识别信号

在扫描论文时注意以下关键词：
- "超声清洗" / "ultrasonic" → 标准方法 ✅
- "水溶消解" / "消解法" / "digestion" → 数值偏低 ⚠️
- "刷洗" / "刷子" / "brushing" → 数值偏低 ⚠️
- "滤膜分级" / "分级过滤" / "membrane filtration" → 需确认消解方式

### 4.3 PM 粒径定义差异

不同论文的 "PM10" 定义可能不同：

| 定义方式 | PM10 含义 | 论文 |
|------|------|------|
| <10μm（标准） | 所有 <10μm 的颗粒 | 王琴 2020 等多数 |
| 2.5-10μm（非重叠） | 仅粗颗粒，不含 PM2.5 | Chen Yazhen 2023 |
| 未说明 | 无法判断 | 多数早期论文 |

**处理规则**：
- Chen Yazhen 2023 的 "PM10_paper" = 2.5-10μm → 标准化处理为 PM10_standard = PM10_paper + PM2.5
- 其他论文若未说明分级方式，假定为 <10μm 标准定义，标注 "假定"

## 五、单位陷阱与标准化

### 5.1 常见单位及换算

| 原文单位 | 目标单位 | 换算 | 注意 |
|------|------|------|------|
| mg/cm² | g/m² | **×10** | 2026-06-17修正: 原错误×0.1已修正为×10 |
| μg/cm² | g/m² | **÷100** | 2026-06-17修正: 原错误÷10已修复为÷100 |
| g/m² | g/m² | 不变 | 目标单位 |
| g·m⁻²·crown⁻¹ | — | **不可换算** | 单位树冠投影面积，与叶面积基不兼容 |
| g/leaf | — | **不可换算** | 单叶质量基，与面积基不兼容 |
| g/kg 叶重 | — | **不可换算** | 质量基，与面积基不兼容 |

### 5.2 不可纳入的单位类型

- **g·m⁻²·crown⁻¹**（单位树冠投影面积）：晏增 2021 使用，与叶面积数据不可比
- **g/leaf**（单叶滞尘量）：依赖叶面积大小，不同物种间不可比
- **g/plant**（单株滞尘量）：Sheng 2024 有 337.141 g·m² 的单株值，不可比
- **g/kg**（单位叶质量）：质量基，需叶面积数据换算

### 5.3 已确认正确的换算

- `mg/cm² × 10 = g/m²`（2026-06-17修正：原错误×0.1已修正为×10）
- `μg/cm² ÷ 100 = g/m²`（2026-06-17修正：原错误÷10已全部修复为÷100）

## 六、数据录入与去重

### 6.1 提取脚本模式

每篇新论文写一个独立提取脚本，遵循统一模式：

```python
# extract_<author>_<year>.py
# 1. 定义 FIELD_NAMES（与 dataset.csv 一致）
# 2. add() 函数填充每条记录
# 3. 输出到 new_records_<author>_<year>.csv
# 4. 计算验证（如年均值 = 四季均值）
# 5. print 摘要供人工复核
```

**好处**：可复现、可审计、可更新。

### 6.2 去重规则

去重键：(`plant_species`, `city`, `functional_zone`, `reference`)

合并前必须检查新记录是否已存在于主数据集：
```python
existing_keys = set(zip(main['plant_species'], main['city'], 
                        main['functional_zone'], main['reference']))
new_records = [r for r in new if key(r) not in existing_keys]
```

### 6.3 数据补录标注

每条记录在 `notes` 字段中必须包含：
- 原始数据来源（"正文提取"/"表格读取"/"年均值(四季均值)"）
- 如果 PM10/PM2.5 为后补录（非同一论文原始数据），标注 `[PM补录]`
- 任何方法学注意事项（如水溶消解法、柱状图读数）
- 四季原始值（如适用），方便后续核查

### 6.4 环境背景值处理

- 论文原文有环境 PM 数据的 → 直接使用
- 论文原文没有的 → 使用该城市该年份官方年均值，**在 notes 中标注来源**
- 同一论文的所有记录应使用一致的环境背景值

## 七、数据质量常见问题

### 7.1 已发现并修正的错误

| 错误 | 发现方式 | 修正 |
|------|------|------|
| 红花檵木 TSP 4.96→2.83 | 原文正文明确指出，柱状图读数偏差 | 以正文为准 |
| 海桐采样日期 2018→2017 | 论文发表时间逻辑矛盾 | 修正为 2017 |
| 高丹丹记录 days_after_rain=7→4 | 原文明确"雨后 4 天" | 全部修正 |
| PM 标签恢复后的数学验证 | 三级 PM 之和 ≠ TSP | 重新对齐行号 |

### 7.2 待修正的已知问题

- **功能区命名不统一**：`交通区` vs `交通干道`，`清洁区` vs `公园清洁区`，`街道绿地` vs `新兴区`——论文写作前必须批量统一
- **麦冬/沿阶草混用**：中文文献常混用两个近缘种，需逐条标注实际鉴定种
- **"城市混合"标签滥用风险**：部分记录标注为"城市混合"但原论文可能是明确的道路/公园数据

### 7.3 异常值处理

遇到异常高/低的值时，必须：
1. 返回原文确认数值
2. 检查测量方法是否不同
3. 检查单位是否有误
4. 在 notes 中标注异常原因
5. 不要静默删除——异常值可能包含重要信息

**案例**：
- 山麦冬 69.039 g/m²（Sheng 2024）：高于平均值 10-50 倍，需验证
- 海桐 PM2.5 = 0.0004 g/m²（Chen Yazhen 2023）：极低，水溶消解法可能对蜡质层洗脱不足

## 九、论文写作前检查清单

- [ ] 功能区命名统一（交通区→交通干道，清洁区→公园清洁区，删除非标准名称）

- [ ] 麦冬/沿阶草逐条核实拉丁名
